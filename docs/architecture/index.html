Operating rules

How future AI sessions should initialize

Read the docs first, summarize the current state, then make the smallest possible change. The docs are the source of truth.

Startup order

  1. Read AGENTS.md
  2. Read README.md
  3. Read docs/index.html
  4. Read the latest session note and commit review
  5. Read docs/prompts/index.html
  6. Read the relevant lesson page
  7. Summarize what was read before touching files

Change order

  1. Update docs first.
  2. Keep the lesson slice tiny.
  3. Explain every SQL statement in plain English.
  4. Finish with a session note and commit review.

Reference policy

When PostgreSQL behavior needs confirmation, use the official documentation as the primary source. This repo teaches from first principles, but it should anchor technical claims in the real upstream docs.
