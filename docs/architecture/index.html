Architecture Decisions

Decision: Static HTML docs are the source of truth.

Why: The site should be readable locally without a build step.

Decision: Curriculum follows a strict dependency order.

Why: Postgres concepts build on each other, especially queries, modeling, and transactions.

Decision: pgAdmin is the local learning interface.

Why: The user already has pgAdmin installed and wants practical database work.

Decision: Prompt history is logged in docs/prompts.

Why: Each session should preserve the instructional context that created it.
