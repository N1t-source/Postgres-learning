Layer State Focus Notes
1 Visible Foundations Session 01: relation theory, psql, first table, INSERT, SELECT.
2 Visible Internals Pages, heap, b-tree, planner, statistics, EXPLAIN, scans.
3 Visible Concurrency MVCC, locks, isolation, deadlocks, snapshots, visibility.
4 Visible Schema Design Normalization, foreign keys, indexes, partial indexes, constraints.
5 Visible Operations Backup, restore, roles, GRANT, pooling, PITR concepts.
6 Visible Performance VACUUM, autovacuum, bloat, planner, slow queries, index-only scans.
7 Visible Production Migrations, replication, monitoring, read replicas, pitfalls.

Why prerequisites still matter

Each later layer depends on the mental model built by the earlier one. The learner should not be shown mechanics before the foundation exists to interpret them.

What comes first

Session 01 establishes relation thinking, the psql workflow, and the first table. That is the entry point for the whole bookstore project, even though the rest of the sequence is now visible.
