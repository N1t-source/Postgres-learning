Curriculum map

40-session progression

The syllabus shows the dependency order for the whole course. Later phases stay locked conceptually until earlier ones make sense.

PhaseSessionsWhat it teachesBoundary example
Foundations01-05Learn the problem a database solves before writing any SQL.Why databases exist → What PostgreSQL is and how psql fits
SQL Foundations06-10Learn the query shape before manipulating rows.SELECT, FROM, and aliases → UPDATE and DELETE
PostgreSQL Basics11-15Learn the toolchain, object lifecycle, and type system.Creating a database and inspecting it → serial vs identity and safe dropping
Data Modeling16-20Turn requirements into related tables with clean boundaries.Entities and attributes from country requirements → Normalization and naming conventions
Constraints21-24Protect the data before relying on it for analysis.PRIMARY KEY and NOT NULL → Integrity failure cases
Joins25-28Use foreign keys and join logic to read across tables.INNER JOIN → CROSS JOIN and self joins
Aggregation29-32Reduce many rows into counts and totals.COUNT and SUM → HAVING and summary reports
Mock Data and Seeding33-34Build reliable development data before production data exists.Seed files and fake data → CSV import and reset scripts
Testing35-36Prove inserts, joins, and constraints behave correctly.Testing inserts and constraints → Testing joins and query results
Transactions37-38Make related writes succeed or fail together.BEGIN, COMMIT, and ROLLBACK → Atomicity and consistency
Indexes and Performance39-40Learn when indexes help and how EXPLAIN explains the plan.Indexes and B-tree basics → EXPLAIN, EXPLAIN ANALYZE, and query planning

Dependency rule

Every later lesson depends on the mental model created by earlier lessons. We do not teach performance before query shape, or joins before foreign keys.

Completion rule

Each session is a tiny slice that fits into one commit and one review. If the quiz score is below the target, the lesson repeats instead of moving on.
