Foundations Session 03

Primary keys and row identity

Show why each row needs a stable identity in the countries table.

Previous Back to session index Next

Learning objective

Show why each row needs a stable identity in the countries table.

What this lesson protects

Without a primary key, updates and relationships become fragile.

Concept explanation

A primary key makes one row distinct from every other row, even when names look similar.

Underlying database or SQL idea

Learn the problem a database solves before writing any SQL.

PostgreSQL-specific explanation

PostgreSQL enforces primary keys so duplicate identities cannot sneak in unnoticed.
