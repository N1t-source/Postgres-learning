Constraints Session 23

Foreign keys and referential actions

Connect tables with foreign keys and explain delete behavior.

Previous Back to session index Next

Learning objective

Connect tables with foreign keys and explain delete behavior.

What this lesson protects

Without foreign keys, orphan rows appear easily.

Concept explanation

Foreign keys keep child rows pointed at valid parent rows.

Underlying database or SQL idea

Protect the data before relying on it for analysis.

PostgreSQL-specific explanation

PostgreSQL can block, cascade, or null out related rows depending on the rule you choose.
