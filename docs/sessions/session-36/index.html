Testing Session 36

Testing joins and query results

Test that join queries return the rows you expect.

Previous Back to session index Next

Learning objective

Test that join queries return the rows you expect.

What this lesson protects

Without query tests, a harmless refactor can change result shape.

Concept explanation

A join test should prove both the happy path and the missing-match path.

Underlying database or SQL idea

Prove inserts, joins, and constraints behave correctly.

PostgreSQL-specific explanation

PostgreSQL joins are predictable once the keys and cardinality are understood.

Lab objective

Check join results for countries with cities and countries without cities.

Step-by-step instructions

  1. Read the concept once more and explain it in one sentence.
  2. Open the session notes and identify the exact file or SQL change this lesson expects.
  3. Write or inspect the smallest possible example for the country model.
  4. Run the SQL commands in the order shown below.
  5. Check the result and compare it to the learning objective.

Expected files changed

  • docs/quizzes/index.html

SQL commands to run
