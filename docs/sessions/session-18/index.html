Data Modeling Session 18

One-to-many relationships

Model one country-to-many cities or regions relationships.

Previous Back to session index Next

Learning objective

Model one country-to-many cities or regions relationships.

What this lesson protects

Without one-to-many modeling, duplicated parent facts spread everywhere.

Concept explanation

One parent row can be referenced by many child rows when the relationship is naturally hierarchical.

Underlying database or SQL idea

Turn requirements into related tables with clean boundaries.

PostgreSQL-specific explanation

Foreign keys make one-to-many relationships explicit and enforceable in PostgreSQL.
