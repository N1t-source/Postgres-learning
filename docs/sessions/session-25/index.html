Joins Session 25

INNER JOIN

Use INNER JOIN to combine related country and city rows.

Previous Back to session index Next

Learning objective

Use INNER JOIN to combine related country and city rows.

What this lesson protects

Without the join mental model, multi-table reports look mysterious.

Concept explanation

INNER JOIN keeps only rows with matching values on both sides.

Underlying database or SQL idea

Use foreign keys and join logic to read across tables.

PostgreSQL-specific explanation

PostgreSQL joins tables by comparing the key columns you specify in the ON clause.

Five questions before the lab

Question 1

Why should the country and city tables be joined instead of repeating country names in every city row?

  • Because joins always make queries faster.
  • Because city rows need more columns.
  • The join preserves one source of truth for the country facts.
  • Because foreign keys are only a naming convention.
Answer check

C. The join preserves one source of truth for the country facts.

Question 2

What is the most likely risk if we skip inner join?

  • Nothing changes because PostgreSQL fills the gap automatically.
  • The later model will have a missing rule or a weak mental model.
  • All joins become invalid immediately.
  • The database engine stops working.
Answer check

B. The later model will have a missing rule or a weak mental model.
