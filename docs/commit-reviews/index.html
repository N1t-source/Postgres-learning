Commit reviews

What each commit should record

One session equals one commit, and the commit review captures the change as a learning checkpoint.

Review fields

  1. Lesson number and title
  2. Files changed
  3. Concept learned
  4. Commands run
  5. Verification result
  6. What we understand now
  7. What we are not allowed to do yet

Why this matters

Commit reviews keep the learning trail understandable after the code and docs have moved on. They turn each change into a reviewable checkpoint instead of a vague history line.
